# Классификация радиосигналов инопланетных цивилизаций

Проект по заданию МПОШ (продуктовый сектор, раздел ИТ). Веб-приложение на Django: авторизация (администратор и пользователь), загрузка тестового набора данных, просмотр аналитики по обученной модели.

## Что сделано

- **Восстановление меток классов** — в наборе данных метки повреждены (строки). В модуле `ml/restore_labels.py` они приводятся к целым числам 0, 1, 2, ...
- **Обучение модели** — скрипт `ml/train_dataspere.py` для Яндекс DataSphere: загрузка Data.npz, восстановление меток, обучение нейросети, сохранение модели в формате .h5 и JSON для графиков.
- **Сайт на Django** — авторизация, две роли (администратор и пользователь), данные в СУБД (SQLite по умолчанию).

### Администратор
- Вход по логину и паролю.
- Единственная возможность: создание новых пользователей (поля: Имя, Фамилия, логин, пароль).

### Пользователь
- Загрузка тестового набора в формате .npz (массивы test_x, test_y).
- Просмотр аналитики: график точности на валидации по эпохам, диаграмма количества записей по классам в обучении, точность по записям теста, топ-5 классов в валидации.
- Графики строятся с помощью **Matplotlib** (из разрешённых материалов). Масштабирование — через масштаб страницы или открытие изображения в полном размере.
- На странице аналитики отображаются точность и потери на загруженном тестовом наборе.

## Используемые технологии (из списка полезных ссылок)

- **Django** — веб-фреймворк, авторизация, СУБД (SQL для Python через ORM).
- **Python 3** — язык разработки.
- **TensorFlow и Keras** — обучение и загрузка нейросети (модель .h5).
- **NumPy** — работа с данными .npz и массивами.
- **Matplotlib** — построение графиков и диаграмм на странице аналитики.
- Обучение в **Yandex Cloud** (DataSphere) по инструкции организаторов.

## Где лежит manage.py

Файл **manage.py** находится в папке **backend/** (не в корне проекта). Все команды Django выполняются из папки backend:

```
cd backend
python manage.py migrate
python manage.py runserver
```

## Как запустить

1. Установить зависимости:
   ```
   pip install -r requirements.txt
   ```

2. Перейти в папку backend и применить миграции:
   ```
   cd backend
   python manage.py migrate
   ```

3. Создать администратора для первого входа:
   ```
   python manage.py create_admin
   ```
   Будет создан пользователь: логин **admin**, пароль **admin** (роль — администратор).

4. Запустить сервер:
   ```
   python manage.py runserver
   ```
   Открыть в браузере: http://127.0.0.1:8000/

5. Войти как admin, создать пользователя через «Создать пользователя». Выйти и войти под созданным пользователем — станут доступны загрузка файла и аналитика.

## Обучение модели в DataSphere

1. Скачать Data.npz с https://disk.yandex.ru/d/BA4oJb0BwaABxg и положить в корень проекта (рядом с папкой `ml`).
2. В DataSphere открыть проект, загрузить код репозитория и Data.npz.
3. Запустить:
   ```
   python ml/train_dataspere.py
   ```
4. После обучения появятся файлы в папке `ml/`:
   - `model.h5` — модель (положить в репозиторий и в облачную папку);
   - `training_history.json`, `train_class_counts.json`, `valid_top5.json` — для графиков на сайте.

В репозитории уже лежат примеры этих JSON (с демо-данными), чтобы аналитика отображалась до первого обучения.

## Unit-тесты

- Тесты Django (авторизация, создание пользователя, профиль, аналитика):
  ```
  cd backend
  python manage.py test users main
  ```
- Тесты восстановления меток (нужен numpy):
  ```
  pip install numpy
  python -m unittest tests.test_restore_labels -v
  ```

## Структура проекта

- `backend/` — Django-проект (config, приложения users и main).
- `ml/` — восстановление меток и скрипт обучения для DataSphere.
- `data/` — здесь можно положить Data.npz (путь в скрипте обучения можно поменять).
- `tests/` — тесты для модуля ml.

Для сдачи: архив репозитория, модель в формате .h5, дамп БД, скриншоты интерфейса — в облачную папку по инструкции организаторов.
