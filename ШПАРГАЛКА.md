# Шпаргалка по кейсу — Классификация радиосигналов

## Запуск сайта
```
cd backend
python manage.py runserver
```
Открыть: http://127.0.0.1:8000/
Логин admin / пароль admin → создать пользователя → войти под ним → загрузить Data.npz

---

## Данные
- Файл: Data.npz (488 МБ)
- Обучение: 1200 записей, Валидация: 400 записей
- Каждая запись: 80 000 замеров яркости звезды (кривая блеска)
- Классов: 20 планетных систем (Kepler-62e, Gliese_163_c, K2-155d и др.)
- Метки повреждены: строки вида "MD5хэш(32 символа) + название планеты"
- restore_labels.py отрезает хэш и использует только название планеты

---

## Модель
- Тип: MLP (полносвязная нейросеть)
- Архитектура: Input(80000) → Dense(256,relu) → Dropout(0.3) → Dense(128,relu) → Dropout(0.3) → Dense(802, softmax)
- Оптимизатор: Adam
- Функция потерь: sparse_categorical_crossentropy
- Эпох: 30, Batch size: 32
- Обучение: Yandex DataSphere (GPU)
- Точность на валидации: ~8% (случайное угадывание = 5% = 1/20, т.е. в 1.6 раза лучше)

---

## Почему такой выбор
| Что | Почему |
|-----|--------|
| MLP (не CNN/LSTM) | Мало данных, MLP проще и быстрее |
| Dropout(0.3) | Защита от переобучения |
| Adam | Адаптивный lr, сходится быстро |
| softmax | Многоклассовая классификация |
| 30 эпох | После 20+ точность не растёт |
| SQLite | Достаточно для олимпиады, встроен в Django |
| Matplotlib | Разрешён организаторами, рендер на сервере |

## Частые вопросы
**Почему точность ~8%?** — 20 классов, 1200 записей = 60 примеров/класс. Случайное = 5%. Модель даёт 8% — лучше случайного, но сигналы очень зашумлены.

**Как работает загрузка файла?** — Django сохраняет .npz → NumPy читает массивы → модель делает предсказания → результат в БД → графики.

**Как строятся графики?** — Matplotlib рендерит PNG в памяти на сервере, браузер получает через `<img>`.

**Зачем restore_labels?** — Метки в датасете — строки с мусором, нужно привести к числам 0..N единым словарём для train и valid.
